## Reporting a Vulnerability

Please report security issues privately and do not disclose them publicly before the maintainers have had time to review and respond.

When reporting, include:

- A short description of the issue.
- The affected area of the application.
- Clear reproduction steps.
- Any proof-of-concept details that help validate the finding.
- The impact you believe the issue could have.

If a private GitHub security advisory or private issue channel is available for this repository, use that first. Otherwise, contact the core maintainers through the repository’s normal maintainer contact path and request a private security conversation.

## Disclosure Process

1. The report is acknowledged privately.
2. The maintainers reproduce and assess the impact.
3. A fix is prepared and reviewed.
4. The reporter is notified when a release or patch is ready.
5. Public disclosure happens only after the fix is available, unless a different timeline is mutually agreed.

## Supported Versions

| Version | Status |
| --- | --- |
| Current main branch | Supported for active security review and fixes |
| Public tagged releases | None published at the time of writing |

If tagged releases are published later, the supported version policy should be updated to match the release cadence.
